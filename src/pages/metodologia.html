Metodologia

Transparência científica e técnica sobre fontes, tratamento e limitações dos dados.
1. Fontes de Dados Oficiais

DATASUS / SIH-SUS — Sistema de Informações Hospitalares do SUS.

Coleta de internações hospitalares por doenças do aparelho respiratório (CID-10 J00–J99) para todos os municípios de Santa Catarina, na série histórica disponível no dataset atual, agrupadas mensalmente por município de residência do paciente. Acesso via tabnet.datasus.gov.br e arquivos DBC via biblioteca pysus.

INMET — Instituto Nacional de Meteorologia.

Dados meteorológicos das estações automáticas de SC: temperatura média, mínima, máxima, umidade relativa e precipitação. Acesso via API pública apitempo.inmet.gov.br, com preservação do último dataset local válido quando a coleta oficial vem incompleta. Total de 20 estações automáticas identificadas em SC.

Open-Meteo — previsão meteorológica de 7 dias usada na aba de previsão, atualizada pela API pública api.open-meteo.com.

2. Associação Município–Estação INMET

Cada município é associado à estação INMET mais próxima geograficamente pela fórmula de Haversine (distância geodésica em km):

d = 2R · arcsin(√[ sin²(Δlat/2) + cos(lat₁)·cos(lat₂)·sin²(Δlon/2) ])

O uso da estação mais próxima é informado nos detalhes de cada município. Municípios sem estação própria utilizam a estação mais próxima disponível. A distância é reportada nos detalhes do município como indicador de limitação metodológica. Municípios com estação a mais de 50km têm representatividade climática reduzida.

3. Correlação de Pearson

Mede a força e direção da associação linear entre temperatura média mensal e quantidade de internações por município, calculada sobre a série histórica completa disponível no dataset.

r = Σ[(xᵢ − x̄)(yᵢ − ȳ)] / √[ Σ(xᵢ − x̄)² · Σ(yᵢ − ȳ)² ]

Correlações negativas indicam que temperaturas menores estão associadas a maior número de internações — padrão típico de doenças respiratórias sazonais no Sul do Brasil. Valores próximos a −1 indicam associação negativa forte.

Faixa de rInterpretação
−1,00 a −0,70Associação negativa forte
−0,70 a −0,40Associação negativa moderada
−0,40 a +0,40Associação fraca ou nula
+0,40 a +1,00Associação positiva moderada a forte
4. Information Value (IV)

Métrica adaptada do contexto de scorecard de crédito (Weight of Evidence) para avaliar o poder preditivo da temperatura sobre as internações. A temperatura é dividida em 4 bins pelos quartis (Q1, Q2, Q3) da distribuição histórica de cada município.

IV = Σ (Dist_Eventos − Dist_Não_Eventos) × WoE  |  WoE = ln(Dist_Eventos / Dist_Não_Eventos)

Onde "Eventos" são meses com internações acima da mediana histórica e "Não Eventos" abaixo.

IVPoder preditivo
< 0,02Negligenciável
0,02 – 0,10Fraco
0,10 – 0,30Médio
> 0,30Forte
5. Estatísticas Descritivas

Média: média aritmética das internações mensais no período.

Mediana: valor central da distribuição de internações mensais.

Desvio padrão: dispersão das internações mensais em torno da média.

Tendência temporal: coeficiente angular de regressão linear simples sobre a série cronológica de internações mensais. Positivo = tendência de crescimento; negativo = redução.

Variação percentual anual: comparação entre o total de internações de anos consecutivos.

8. Mecanismo de Coleta Resiliente

O pipeline Python implementa coleta resiliente e registra explicitamente quando uma fonte oficial não retorna uma atualização completa:

Etapa 1: API oficial (TabNet DATASUS / API INMET REST).

Etapa 2: Download de arquivos públicos (DBC via pysus / BDMEP).

Etapa 3: manutenção do último dataset local válido, sem sobrescrever os dados quando a coleta vem incompleta.

Toda execução é registrada em data/status_dados.json e data/logs.json com timestamp, fonte utilizada, status e número de registros coletados.

9. Automação — GitHub Actions

O workflow .github/workflows/atualizar-dados.yml executa automaticamente no primeiro dia de cada mês (03:00 UTC). Realiza a coleta, processamento, commit automático e publicação via GitHub Pages. Execução manual disponível pelo painel do GitHub Actions.

10. Limitações do Estudo

• Dados de internação por município de residência, não de ocorrência — pode introduzir viés em municípios sem hospital próprio.

• Associação de municípios a estações distantes pode reduzir a representatividade climática local.

• Correlação não implica causalidade; outros fatores (poluição, densidade, acesso à saúde) influenciam as internações.

• O sistema utiliza dados históricos e previsão meteorológica de 7 dias; não realiza previsão epidemiológica nem diagnóstico médico.

• Municípios pequenos podem apresentar maior variância estatística por volume reduzido de internações.

• A disponibilidade das APIs públicas pode variar entre execuções.

⚠️ Segurança Jurídica e Metodológica: Este sistema utiliza dados públicos oficiais para fins acadêmicos, científicos e informativos. Os resultados representam associações estatísticas e não estabelecem causalidade. Este sistema não realiza diagnóstico médico e não substitui orientação médica, epidemiológica ou decisões governamentais. Eventuais inconsistências podem decorrer das limitações das bases públicas utilizadas.

Metodologia
Transparência científica e técnica
1. Fontes de Dados
DATASUS / SIH-SUS: Dados de internações hospitalares por doenças respiratórias (CID-10 J00–J99) do Sistema de Informações Hospitalares do SUS para o estado de Santa Catarina, período 1990–2026, agrupados mensalmente por município de residência do paciente. Fonte: tabnet.datasus.gov.br e arquivos DBC via biblioteca pysus.

INMET: Dados meteorológicos das estações automáticas de Santa Catarina, coletados via API pública (apitempo.inmet.gov.br). Variáveis: temperatura média, mínima, máxima, umidade relativa e precipitação.
2. Associação Município–Estação
Cada município é associado à estação meteorológica do INMET mais próxima geograficamente, calculado pela fórmula de Haversine (distância geodésica em km):

d = 2R · arcsin(√[sin²(Δlat/2) + cos(lat₁)·cos(lat₂)·sin²(Δlon/2)])

Quando o município não possui estação própria, a estação mais próxima é utilizada e a distância é informada nos detalhes do município como indicador de limitação metodológica. SC conta com 20 estações INMET identificadas neste estudo.
3. Correlação de Pearson
Mede a associação linear entre temperatura média mensal e quantidade de internações. Calculada para cada município individualmente sobre a série histórica completa disponível.

r = Σ[(xi − x̄)(yi − ȳ)] / √[Σ(xi − x̄)² · Σ(yi − ȳ)²]

Correlações negativas indicam que temperaturas menores estão associadas a maior número de internações — padrão típico de doenças respiratórias sazonais. Valores próximos a −1 indicam associação negativa forte; próximos a +1, positiva forte.
4. Information Value (IV)
Métrica adaptada do contexto de scorecard de crédito para avaliar o poder preditivo da temperatura sobre as internações. A temperatura é dividida em 4 bins por quartis, e o IV é calculado como: IV = Σ (dist_eventos − dist_não_eventos) × WoE, onde WoE = ln(dist_eventos / dist_não_eventos).

Classificação: <0,02 = Negligenciável; 0,02–0,10 = Fraco; 0,10–0,30 = Médio; >0,30 = Forte.
5. Classificação de Risco e Vulnerabilidade
Risco climático baseado na temperatura média histórica: ≥18°C = Baixo; 15–18°C = Moderado; 12–15°C = Alto; <12°C = Muito Alto.

Vulnerabilidade combina correlação, IV e temperatura média em score composto (0–6 pontos): ≥5 = Muito Alta; ≥3 = Alta; ≥1 = Moderada; 0 = Baixa.
6. Limitações
• Os dados de internação são por município de residência, não de ocorrência, podendo introduzir viés em municípios sem hospital próprio.
• A associação de municípios a estações distantes pode reduzir a representatividade climática local.
• Correlação não implica causalidade; outros fatores (poluição, densidade populacional, acesso à saúde) influenciam as internações.
• O sistema utiliza dados históricos; não realiza previsões meteorológicas.
• Municípios pequenos podem apresentar flutuações estatísticas maiores por volume reduzido de internações.
7. Atualização Automática
Os dados são atualizados mensalmente via GitHub Actions (workflow atualizar-dados.yml), que executa os scripts Python de coleta e processamento, realiza commit automático dos arquivos atualizados e publica via GitHub Pages. O fluxo é resiliente com 3 etapas: API oficial → download público → último dataset local válido.
Sobre o Projeto
Contexto acadêmico e social
Por que este projeto importa?
Este projeto transforma dados públicos em conhecimento acessível para apoiar estudos acadêmicos, pesquisas e ações preventivas. A plataforma demonstra como a análise de dados pode auxiliar na compreensão de padrões relacionados à saúde pública e às condições climáticas, fortalecendo o uso de evidências na tomada de decisão.
Contexto Acadêmico

Curso: Sistemas de Informação — UDESC


Objetivo: Demonstrar integração de múltiplas disciplinas: Sistemas de Informação, Engenharia de Software, Análise e Ciência de Dados, Estatística, Visualização de Dados, Saúde Pública e Automação.


Diferenciais técnicos: Coleta resiliente com status explícito das fontes, pipeline de dados automatizado via CI/CD, cálculo de IV adaptado, mapa interativo com GeoJSON municipal, e zero dependência de banco de dados ou backend hospedado.

Regressão Linear Múltipla
O modelo combina temperatura, umidade e sazonalidade (via funções seno/cosseno do mês) para estimar internações futuras. Treinado por município sobre a série histórica 1990–2026.

⚠️ As estimativas são projeções estatísticas baseadas em padrões históricos, não previsões epidemiológicas. Não substituem análise profissional.
